    ```typescript
    /**
     * BrandLogo — Leasétic light/dark logo lockup.
     *
     * Renders TWO <img> tags side-by-side; the brand-logo CSS picker rules in
     * app/globals.css (added by Plan 11-01) hide whichever variant does not
     * match the current <html data-theme> attribute. Zero JavaScript — rides
     * the no-flash inline script that sets data-theme before first paint.
     *
     * Server component by D-09 / UI-SPEC §6.1.
     */
    export interface BrandLogoProps {
      /** Pixel width. Defaults to 190 (sidebar brand-row width per UI-SPEC §6.3). */
      width?: number;
      /** Pixel height. Defaults to 32 (1192×200 viewBox aspect at w=190 ≈ h=31.9). */
      height?: number;
      /** Required for non-decorative usage. If omitted, both <img> render alt="". */
      alt?: string;
      /** Optional class appended to outer <span>'s className. */
      className?: string;
    }

    export function BrandLogo({
      width = 190,
      height = 32,
      alt = '',
      className = '',
    }: BrandLogoProps) {
      const wrapperClass = `brand-logo${className ? ' ' + className : ''}`;
      return (
        <span
          className={wrapperClass}
          style={{ display: 'inline-block', lineHeight: 0 }}
        >
          {/* eslint-disable-next-line @next/next/no-img-element -- intentional: CSS picker + zero-JS theme switch per CONTEXT D-09 */}
          <img
            className="brand-logo-light"
            src="/logo-light.svg"
            alt={alt}
            width={width}
            height={height}
          />
          {/* eslint-disable-next-line @next/next/no-img-element -- intentional: CSS picker + zero-JS theme switch per CONTEXT D-09 */}
          <img
            className="brand-logo-dark"
            src="/logo-dark.svg"
            alt={alt}
            width={width}
            height={height}
          />
        </span>
      );
    }
    ```

    Note on `eslint-disable-next-line @next/next/no-img-element`: Next.js 16's default eslint config flags `<img>` and recommends `<Image>` (next/image). For the brand-logo theme picker, `next/image`'s optimization layer breaks the CSS-picker approach (`<Image>` renders a `<span>` wrapper that complicates the `.brand-logo-light` / `.brand-logo-dark` class selector targeting). Use raw `<img>` with the eslint-disable comment per CONTEXT D-09 explicit "Zero JS" mandate.

Do NOT add a `'use client'` directive — this is a server component (no state, no event handlers). Do NOT consume `next/image` — see eslint-disable rationale above.
