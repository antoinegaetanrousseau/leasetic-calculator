### Plumbing Chain

```
Partner clicks Duplicate <Link href="/proposals/new?duplicate={id}">
  └→ page.tsx: await searchParams → sp.duplicate
       └→ getProposalById(sp.duplicate)
            ├─ source NOT found / NOT owned / deleted → prefill = { partnerName } (silent fallback)
            └─ source found + owned + active → prefill = { ...source.inputs, partnerName }
  └→ <DuplicatePrefillToast lang={lang} /> (mounts, fires toast.info once, strips URL)
  └→ <ProposalFormProvider prefill={prefill}> — all 14 fields populated
       └→ <ProposalForm> — useState lazy captures duplicatedFromId from window.location.search
            └→ onSubmit: POST /api/proposals body includes duplicatedFromId
                 └→ submitProposal() → createProposal({ duplicatedFromId }) → INSERT row
```

### What Is NOT Copied from Source (PROP-23 immutability)

- `paramsSnapshot` — freshly read from current `global_params` at submit time
- `computed` — re-derived from current `global_params` via `computeLoyer()`
- `pdfBlobKey`, `pdfSha256`, `pdfSizeBytes`, `pdfGeneratedAt` — regenerated from fresh render
- `idempotencyKey` — fresh `crypto.randomUUID()` from `useState` in `ProposalForm`
- `lcRef` — freshly generated by `generateLcRef()` in `submitProposal`

Only `source.inputs` (the 14 user-typed fields) is copied, and even those are editable before submit.

## Edge Cases Handled

| Case | Behavior |
|------|----------|
| `?duplicate=<owned-id>` (active) | All 14 fields pre-filled from source.inputs |
| `?duplicate=<unowned-id>` | Silent fallback to default prefill (partnerName only) |
| `?duplicate=<deleted-id>` | Silent fallback (deletedAt guard in ownership check) |
| `?duplicate=<malformed>` | `getProposalById` returns null → silent fallback |
| `?duplicate=<missing-id>` | `getProposalById` returns null → silent fallback |
| URL re-visited after redirect | No ?duplicate= in URL → no toast, no prefill |

## Known Trade-off

`DuplicatePrefillToast` mounts (and fires the toast) whenever `sp.duplicate` is non-empty in the URL — including when the source row was not found or not owned (in which case prefill is empty but the toast still fires). This is an accepted UX trade-off per the plan's threat model (T-08-13-01): the toast message is generic ("Fields pre-filled…") and does not confirm whether prefill actually succeeded, so no information about another user's data is leaked.
