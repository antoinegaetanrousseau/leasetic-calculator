#### 3.1.7 Validity logic (derived, not stored)

`<ValidityChip>` variant is derived at render time:

```
const expiresAt = createdAt + (validityDays * 86400000); // ms
const variant = (now() < expiresAt) ? 'active' : 'expired';
```

Expiration is recomputed on every page render. NOT cached, NOT denormalized into the row. (Phase 8 doesn't add a `validity_expires_at` column — derivation is cheap and avoids data drift.)

### 3.3 PDF visual layout (the rendered document)

The rendered single-page PDF produced by `src/lib/pdf/Document.tsx` via `@react-pdf/renderer`. Per D-A1, this is a **professional layout, NOT a v10-equivalent literal port.** v10's `@media print` CSS uses Grid + multi-page; `@react-pdf/renderer` uses a flexbox-ish subset and supports neither (PITFALLS §4.3).

Per D-A2, every PDF is single-language (FR or EN) — language captured at gen time from `session.user.language` and stored in `proposals.language text NOT NULL`. The renderer branches on `props.language` to produce the correct dictionary.

Per D-A3, the footer is **minimal: LC ref + creation date + page number ONLY.** No mentions légales, no signature line.

- Base color: `#1a2832` (deep navy-ink, matches the dark-mode PDF surface fallback already in `globals.css` line 103).
- Background: pure white (`#ffffff`), NEVER `--paper` — PDF parity invariant per Phase 5 globals.css `[data-pdf-surface]` rule.

#### 3.3.14 Locales (PROP-18)

PDF render uses `formatCurrency` / `formatDate` / `formatNumber` from `src/lib/i18n/format.ts` with EXPLICIT `'fr-FR'` or `'en-GB'` locale arguments (Phase 6 D-28). NEVER system defaults. CI byte-determinism gate (PROP-17) catches any drift.

Date format:
- `fr-FR`: "DD/MM/YYYY" (e.g., "09/05/2026")
- `en-GB`: "DD/MM/YYYY" (en-GB matches fr-FR for date format — happy coincidence)

Currency format:
- `fr-FR`: "75 000,00 €" (narrow no-break space U+202F as thousand sep, comma as decimal, euro after)
- `en-GB`: "€75,000.00" (Intl with currency:'EUR' + locale 'en-GB' → "€" prefix, comma thousand sep, dot decimal)

Per Phase 6 D-28 explicit locale lock: NEVER use `Intl.NumberFormat()` no-arg form.

#### 3.3.15 Byte-determinism notes (PROP-17)

The PDF render must produce byte-identical output for byte-identical input. Determinism contract:
- `Font.register` paths are static literals — no env-var injection
- No `Date.now()` calls in the render tree (creation date comes from the proposal row, NOT from Date.now())
- No random IDs (React internal keys are deterministic for a given subtree)
- No locale fallbacks (explicit lang arg always)
- All hex literals — no `var(--*)` resolution
- `@react-pdf/renderer` PDF metadata (Producer / CreationDate / ModDate) is locked via `<Document creationDate={...} modificationDate={...} producer="Leasetic Matrice v1.1" creator="Leasetic Matrice v1.1">` — pass the proposal's `created_at` so the PDF's metadata creation timestamp matches the row (otherwise `new Date()` at render time would drift)

CI fixture: `__pdf-fixtures__/expected.sha256.txt` contains expected hashes for ≥1 fixture proposal × 2 langs = 2 hashes. Drift = CI red. Update workflow: `npm run pdf:update-fixture` regenerates the file when intentional rendering changes happen (e.g., new minor schema_version per D-D3 minor bump).

## 12. Accessibility Floor (NON-NEGOTIABLE)

Every interactive surface declared above MUST satisfy:

- **Keyboard reach:** every action is reachable via Tab; rows are focusable as link targets (Tab → next row); Recently Deleted toggle is a tablist with arrow-key cycling; Search bar focuses naturally; Load More button is a regular button
- **Visible focus:** Phase 5 teal box-shadow ring on every focusable element — never `outline: none` without a `:focus-visible` replacement (rows get a 2px box-shadow ring on focus)
- **ARIA labels:**
  - Search bar input: `aria-label={t('proposal.search.aria', lang)}`
  - Search bar clear button: `aria-label={t('proposal.search.clear', lang)}`
  - Recently Deleted toggle: `role="tablist"`, each button `role="tab" aria-selected="..."`
  - List rows: `<Link>` with `aria-label={`Proposition {clientCo} {lcRef} - {amountFormatted}`}` for screen-reader summary
  - Validity chips: tooltip via native `title` + visible label inside the chip (no aria-only)
  - Action buttons: visible labels (no icon-only buttons — every action button has both icon + label)
  - PDF preview embed: `aria-label={t('proposal.detail.pdf.preview.aria', lang)}`
- **Live regions:**
  - Sonner toasts already use `role="status"` (Sonner default) — no extra wiring needed
  - Search results count: optional v1.2 enhancement (e.g., "12 propositions trouvées" announced); not in v1.1 scope
- **Color contrast:**
  - Body text on `--surface`: `--ink` (#41423d on #ffffff = 8.6:1 ✓ AAA)
  - Validity-active chip: `--gd` (#129657) on rgba(18,150,87,0.12) tinted bg ≈ #e9f5ee — passes 4.5:1 AA
  - Validity-expired chip: `--gold` (#e08530) on rgba(224,133,48,0.12) tint — passes 4.5:1 AA
  - Deleted chip: `--danger` (#dc2626) on rgba(220,38,38,0.08) tint — passes 4.5:1 AA
  - PDF text: `#1a2832` ink on `#ffffff` = 11.5:1 ✓ AAA (PDF colors are print-grade contrast)
- **Reduced motion:**
  - All transitions wrapped in `@media (prefers-reduced-motion: no-preference)` for the in-app surfaces
  - PDF render itself has no motion (static document)
  - Sonner respects user's prefers-reduced-motion setting (Sonner default)
