Existing listProposalsByUser MUST be updated to filter OUT drafts when args.deleted is false: WHERE `status = 'active'` (or `status != 'draft' AND status != 'deleted'`). Currently it filters only on `deleted_at IS NULL` — that predicate now also matches drafts since drafts have deleted_at NULL. Without this fix, Phase 8's partner-home list would suddenly include drafts. The CONTEXT specifically calls out "the existing `deleted_at IS NULL` query predicates in Phase 8 helpers (proposals.ts:157-162, 213-218) continue to work unchanged" — meaning the deletedAt predicate is correct for the "is it tombstoned" check; the additional `status = 'active'` filter is the new layer that drafts add. Verify this interpretation with the executor reading 12-CONTEXT.md before implementing.

deriveDisplayStatus logic per D-07:
- If row.status === 'deleted' → 'deleted'
- If row.status === 'draft' → 'draft'
- If row.status === 'active':
  - If row.pdfGeneratedAt is null → 'active' (shouldn't happen post-finalize since pdfGeneratedAt is set in same UPDATE; defensive)
  - Compute expiresAt = pdfGeneratedAt + (paramsSnapshot.validityDays || 30) days. If now() > expiresAt → 'expired'. Else 'active'.

The proposals_status_check CHECK constraint blocks invalid status values at the DB layer regardless. |
