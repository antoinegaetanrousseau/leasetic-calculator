## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| auth session create → users.last_login_at write | The session is already authenticated at this point (Better Auth validated the credential); the hook runs server-side with the elevated `db()` connection |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-07-01 | EoP | racy concurrent logins for the same user overwrite last_login_at | accept | Both writes are idempotent — now() is monotonic enough for an "ever logged in" check; the precise value (last login among concurrent sessions) is acceptable noise. Documented in CONTEXT D-11 / security_threat_model guidance T-EoP. |
| T-12-07-02 | Repudiation | last_login_at writes silently fail and the partner is misclassified as 'invited' | mitigate | Best-effort error path logs to console.error (Vercel runtime logs surface this). DB-02 surface tolerates ephemeral lag — the partner who just logged in will be re-classified on their NEXT successful login. Catastrophic loss requires the DB to be unreachable for many consecutive logins — unrealistic given DB availability SLA. |
| T-12-07-03 | Tampering | client-spoofed userId via the hook | mitigate | The hook receives `session.userId` from Better Auth's session object — Better Auth populates this from the validated credential. No client-controllable path reaches the hook with an arbitrary userId. |
| T-12-07-04 | DoS | hook fails fast on the same DB → blocks login surface | mitigate | Try/catch wraps the DB call; login succeeds even on transient DB-write failure. The Better Auth session row was already committed before the after hook runs. |
| T-12-07-05 | Information disclosure | last_login_at exposed to partner-facing surface | accept | Partner home does not surface last_login_at; only admin Liste des partenaires (already gated by requireAdmin()) shows it. Existing exposure via Phase 9 admin list is unchanged. |
</threat_model>
