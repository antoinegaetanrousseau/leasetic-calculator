## 5. Components — Per-Element Specifications

This is the load-bearing section. Each component below has a complete prop signature, layout, state list, and interaction contract. The executor implements from this section without further design ambiguity.

### 5.1 `<WizardActionBar>` (NEW, route-private at `app/(authed)/proposals/new/_components/WizardActionBar.tsx`)

**File:** `app/(authed)/proposals/new/_components/WizardActionBar.tsx` ('use client')
**Renders:** a single `.card` with a horizontal flex row of action controls. Used on all 3 steps; composition varies per step via props.

**Prop signature:**
```ts
export interface WizardActionBarProps {
  /** Step number — drives "← Précédent" visibility (omitted on step 1). */
  currentStep: 1 | 2 | 3;
  /** Draft id, threaded into hrefs for "← Précédent" navigation. */
  draftId: string;
  /** Server-action handler for "Enregistrer comme brouillon". */
  onSaveDraft: () => void;  // wired to a useTransition + server-action call
  /**
   * Primary CTA spec. On steps 1 & 2: { kind: 'link', href, label }.
   * On step 3: { kind: 'action', onClick, label, isSubmitting }.
   */
  primary:
    | { kind: 'link'; href: string; label: string }
    | { kind: 'action'; onClick: () => void; label: string; spinnerLabel: string; isSubmitting: boolean };
  /** Language for the "← Précédent" label only. */
  lang: Lang;
}
```

**Layout (left → right inside the `.card`):**
- Row 1 (single horizontal row):
  - **Conditional** `← Précédent` text-link (omitted entirely when `currentStep === 1`). Uses `<Link>` from `next/link` with `href = /proposals/new/${prevSlug}?draft_id=${draftId}` where `prevSlug = currentStep === 2 ? 'parametres' : 'calcul'`. Text-link style (`color: var(--muted)`, hover `color: var(--ink)`, no underline by default), font-size 14.5px / weight 500. Left edge of the card.
  - **Gap 12px (md)**.
  - `Enregistrer comme brouillon` `.btn-out` ghost button. `type="button"`. `onClick={onSaveDraft}`. Disabled during `isSubmitting`.
  - **`flex: 1` spacer** (pushes the primary CTA to the right).
  - **Primary CTA** — when `primary.kind === 'link'`: `<Link>` styled as `.btn-green` (use `style={{ display:'inline-flex', alignItems:'center' }}` and apply `.btn-green` className to a `<Link>` directly). When `primary.kind === 'action'`: `<button type="submit" className="btn-green" disabled={isSubmitting} onClick={primary.onClick}>`. Right edge of the card.
    - When `isSubmitting === true` (only relevant for step 3's action form): label morphs to `primary.spinnerLabel`, a `<Loader2 size={16} strokeWidth={2}>` icon spins to the LEFT of the label (gap 8px / sm), button gets `aria-busy="true"`, button is disabled. Button bg is `--gd` with `filter: brightness(0.9)`.

**Interaction states:**
- **Idle:** all 3 buttons fully interactive. Primary CTA hover: bg shifts from `var(--gd)` to `#0d7d49` (existing `.btn-green:hover` rule).
- **Saving (after `onSaveDraft` invocation, until redirect resolves):** `Enregistrer comme brouillon` button disabled with `opacity: 0.6` (existing `.btn-out:disabled` rule), primary CTA also disabled to prevent double-submission, `← Précédent` link visually unchanged but its `pointer-events: none` to prevent navigation during save.
- **Submitting (step 3 only, after `onClick={primary.onClick}` invocation, until redirect or error):** primary CTA shows spinner-state per above; all other controls in the bar are disabled (`pointer-events: none` + `opacity: 0.6`).
- **Error (step 3 only, after a finalize failure):** primary CTA re-enables with idle styling; sonner error toast fires; bar returns to idle.

**Focus order (Tab key, idle state):**
1. `← Précédent` link (if present)
2. `Enregistrer comme brouillon` button
3. Primary CTA

**Accessibility:**
- Primary CTA `<button type="submit">` on step 3 lives inside a `<form>` so Enter on focused field submits.
- Spinner-state: button gets `aria-busy="true"`, `aria-live="polite"` region announces the spinner label change for screen readers.
- `← Précédent` link gets explicit `aria-label="Étape précédente"` (FR) / `"Previous step"` (EN) so screen readers don't read just the bare arrow glyph.

**Sticky-footer treatment:** **NOT sticky.** The action bar is a regular `.card` at the bottom of the main content flow (per D-19). It scrolls with the page. Planner may revisit if step 1's expanded `+ Plus de détails` accordion pushes content past viewport (a deferred-idea per `13-CONTEXT.md`).

**Layout:**

'rotate(45deg)' : 'rotate(0)', transition: 'transform 200ms ease-out' }}`) — gap 8px (sm) — label text `+ Plus de détails (facultatif)` (FR) / `+ More details (optional)` (EN).   - When closed: `height: 0`, `overflow: hidden`, `opacity: 0`, content NOT removed from DOM (preserves RHF state across collapses).

  - Transition: `height` and `opacity` animate over **200ms ease-out** (per `13-CONTEXT.md` `<specifics>`).
  - Top margin from the trigger row when open: 16px (lg).
  - Inside the region: the 5 fields are rendered in the order specified in §3 (clientRole → clientSiren → projectDesc → slb → evalParc) using the existing `.fld` rhythm (12px gap between fields per `.fld + .fld { margin-top:12px }`).

**Interaction:**
- Click trigger → toggle `open` state in component (useState) → call `onToggle(newOpen)` server action → server writes `_uiAccordionOpen: newOpen` into draft inputs jsonb so reload preserves expansion.
- Server action is fire-and-forget (no `await`, no toast — purely cosmetic state). If the server write fails silently, the next `updateDraft` from a different action recovers it.
- Focus order through fields when open follows the rendered order. Tab from accordion trigger → first field inside open region (clientRole).

**Animation timing:** 200ms ease-out (CSS `transition: height 200ms ease-out, opacity 200ms ease-out`). The 200ms is intentionally faster than the 300ms "feel sluggish" threshold and slower than the 100ms "instant" threshold — sweet spot for "this is a deliberate UI change". Matches v1.0/v1.1 hover-transition timing (150ms) family.
