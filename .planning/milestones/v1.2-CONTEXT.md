# v1.2 Milestone Context — UX Polish + Proposal Wizard

This file was produced from a design session that sketched all 8 internal-tool surfaces in Figma. The Figma file is the authoritative design contract. Each requirement below maps to a sketched frame in the file.

## Milestone Goal

Elevate the v1.1 functional internal tool into a polished, brand-aligned product:
1. Apply the actual Leasétic brand identity (logo from webflow.io, brand-aligned design tokens)
2. Split the dense single-page proposal flow into a 3-step wizard with persistent draft state
3. Add small UX additions (home metrics, retractable sidebar, status chips throughout, admin nav cards)
4. Move admin partner creation from a modal to a dedicated page

**Layout shells in v1.2:**
- `(public)`: simple centered layout — paper bg + logo above card (login + invite/[token] + reset/[token])
- `(authed)` & `(admin)`: full shell — retractable sidebar + topbar + footer + main paper bg

**Background fill rule (added 2026-05-11 from user adjustment):**

| Layer | Fill | Why |
|-------|------|-----|
| Sidebar | `surface` (white) | Branded chrome zone — pairs with topbar |
| Topbar (page title + ADMIN pill area) | `surface` (white) | Branded chrome zone — pairs with sidebar |
| User menu pill (avatar + name + chevron) | **no fill, border only** | Ghost button pattern — lower visual weight inside the white topbar |
| Main content area | `paper` (#f5f7fc light / #0c121c dark) | Soft canvas for content |
| Hero section (greeting + subtitle + CTA) | **no fill — transparent, inherits paper** | Hero is content, not a card — sits directly on paper bg |
| Stat tiles, form cards, list cards | `surface` (white) | Elevated content containers — visible against paper |
| Action footer cards (wizard back/next buttons) | `surface` (white) | Same as content cards |
| Footer (copyright strip) | `paper` (transparent over main) | No visual chrome — purely informational |

**The principle:** sidebar + topbar = "branded shell" (white). Inside that shell, the main area is paper. Cards float on paper as white elevated surfaces. Hero text sits on paper without its own card. This creates 3 visual layers: shell (white) → canvas (paper) → elevated content (white cards on paper).

Design system tokens:
- Variable collection: `Leasetic / Brand` (23 variables, Light + Dark modes)
- Includes: full brand palette (navy/gd/teal/gold/danger), surface/paper/ink/muted/border (themed), radius scale (6/8/12/16/full), layout dims (sidebar-w 260, topbar-h 64, footer-h 48), `color/brand-mark` (#6DC388 — exact mark color from webflow logo)
- 13 semantic text styles (heading/hero, heading/section, heading/sub, body/large, body/default, label/section-title, label/field, button/default, input/default, chip/default, toggle-pill)
- 2 effect styles: `shadow/card`, `shadow/focus-ring`

## v1.2 Requirements (all confirmed in scope)

### Database / schema

1. **DB-01**: Add `draft` proposal status (enum: `draft | active | expired | deleted`)
   - Enables Save-as-draft button in wizard step 1
   - Required for auto-save between wizard steps
2. **DB-02**: Add `invited` partner account status (alongside `actif | désactivé`)
   - For accounts that have been created but never logged in
   - Surfaces in Liste partenaires as a separate visual chip
3. **DB-03**: Coefficient change history table
   - Columns: `id, changed_at, changed_by_user_id, before_json, after_json, summary`
   - Powers the History sidebar on the Coefficients page

### Routing

4. **ROUTE-01**: Split `/proposals/new` into 3 wizard routes
   - `/proposals/new/parametres` → form for client + project data
   - `/proposals/new/calcul` → calculation result display
   - `/proposals/new/verification` → final review + save
   - Server-side draft persistence between steps (uses DB-01)
   - Stepper component on all 3 — click-back on completed steps only
5. **ROUTE-02**: Dedicated `/[adminSegment]/partners/new` route
   - Replaces the v1.1 modal-based partner creation
   - 3-section form (personal info, company info, invitation message)

### Components (new reusable)

6. **COMP-01**: Stepper component
   - Props: `currentStep: 1|2|3`, `clickableSteps: number[]`
   - States per step: `active`, `pending`, `done` (with green check)
   - Pattern: green circle + label + connector lines between
7. **COMP-02**: Retractable sidebar
   - Two states: expanded (260px) ↔ collapsed (72px)
   - Stored user preference in localStorage
   - Collapsed shows logo + icons only; expanded shows full nav + lang/theme toggles at bottom
   - Logo at top, nav items as buttons with Lucide icons + green-tint active state
8. **COMP-03**: Home metric tile (3 instances)
   - Pattern: label (.ctitle) + big value (heading/section) + sublabel (body/small)
   - Color-coded value: `Ce mois-ci` (green/gd), `Total` (navy), `Brouillons` (gold)
   - Powered by aggregate queries on proposals table
9. **COMP-04**: Admin nav cards (3 instances on admin home)
   - Pattern: 48px icon square (10% opacity bg) + title (heading/sub) + description + "Ouvrir →" link
   - Used for Coefficients / Partenaires / Historique navigation
10. **COMP-05**: Status chip variants
    - 4 variants: `active` (green), `draft / brouillon` (gold), `expired / expirée` (gray-muted), `disabled / désactivé` (red-danger)
    - Used in: proposal list rows, partner table rows
    - Extends existing `.chip-*` classes in globals.css

### Brand assets

11. **ASSET-01**: Add Leasétic logo files to `public/`
    - Source: https://leasetic.webflow.io/ logo at `cdn.prod.website-files.com/.../leasetic-logo.svg`
    - Mark color: `#6DC388` (new `color/brand-mark` token)
    - Wordmark color: `#112C3B` (existing `--navy`)
    - Used in: sidebar (authed/admin), login page (public)
12. **ASSET-02**: Light/dark mode logo variants
    - Light mode: wordmark in `--navy` (provided)
    - Dark mode: wordmark in `--ink` light variant (needs design — request white wordmark SVG)
    - File naming: `logo-light.svg`, `logo-dark.svg`

### Public surfaces (added 2026-05-11 — user slotted login into design contract)

13. **PUB-01**: Apply brand logo + paper background to login page
    - Replace v1.1 text-only "Leasétic" header with the official lockup SVG (ASSET-01)
    - Center logo above the existing form card
    - Use paper (`--paper`) for body background, surface (`--surface`) for the card
    - Existing form structure preserved (no field changes — just visual treatment)
    - Topbar's lang/theme toggles stay on login (different from authed/admin where they moved to sidebar)
14. **PUB-02**: Apply same pattern to `/invite/[token]` and `/reset/[token]`
    - Same `(public)` layout shell: centered logo + card
    - These are not separately sketched but inherit the login pattern
    - Confirms public layout component is reusable across all 3 public routes

### Migration considerations (non-blocking)

- Webflow brand uses near-black for headings; v1.1 uses `--ink: #41423d` (charcoal). Decision: **keep softer ink for internal tool comfort** — flagged but not changing.
- All sketches at 1512px breakpoint (matches what html.to.design captured for `/home`). Mobile not in v1.2 scope per existing PROJECT.md constraint.

## Out of scope for v1.2 (continue deferring)

- OVH production deployment
- Mobile-optimized layout
- Multi-language beyond FR + EN
- Automated browser tests
- Excel export of proposals
- Webhook notifications

## How this gets built

The Figma design contract + 14 requirements above translate to roughly:
- **1 design system phase** (tokens, base components — Stepper, RetractableSidebar, StatusChip, StatTile, AdminNavCard, etc.)
- **1 schema phase** (DB-01, DB-02, DB-03 migrations + tests)
- **1 wizard phase** (ROUTE-01 — three new routes + draft persistence + stepper integration)
- **1 admin polish phase** (ROUTE-02 partner creation page + Coefficients history + admin home dashboard + Liste partenaires status chips)
- **1 brand assets + public surfaces phase** (ASSET-01, ASSET-02, PUB-01, PUB-02 — drop in logo files, wire up across authed/admin sidebars + all 3 public routes)

Final phase decomposition decided by the roadmapper subagent during `/gsd-new-milestone`.
